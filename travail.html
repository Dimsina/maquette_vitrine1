Ici je vais montrer a quoi ressemble le code de mon projet, structuré en MVC

View

L'image de gauche présente une partie du code représentant une des 'view' (le visuel) de mon site de rencontres

C'est ce que l'utilisateur du site voit. Du moins presque, c'est la structure visuelle du site qui est programmée sur ce type de page.

View côté site

Voici donc ce que voit l'utilisateur du site. C'est la traduction du code avec toute la mise en page.

Model

Ici une partie de mon code qui sert de 'model'.

C'est par ce biais que je fais des requêtes a ma base de données(BDD). La BDD est l'endroit où l'on stocke les informations des inscrits. C'est grace a ces informations stockées que l'on peut effectuer les connexions/déconnexions.

Le 'Model' est invisible par les utilisateurs du site. Ici il est codé en PHP et ne s'occupe que du serveur

Le 'Model' est géré par le 'Controller' qui s'occupe de donner les ordres. C'est le 'Controller' qui distribue les actions a faire si les informations données au préalables par le formulaire d'inscription, sont correctes.

Controller

Ici le 'Controller' va vérifier si le surnom et l'email ont bien été entrées lors de la connexion, si oui, alors le controller va commander à l'action 'profil' et donc connecter l'utilisateur a sa page de profil

Autrement dit, sans lui, le script ne se lancera pas, car c'est lui qui fait la gestion d'erreurs (vérifications des infos, complètes ou non, existantes ou non etc...)
